# SPEC-16 — Zod 4 migration + native `z.toJSONSchema` (drop zod-to-json-schema)

- **Status:** Todo · **Pillar:** Future-proofing (deps) · **Wave:** 4 · **Priority:** P=7
- **I**=3 **F**=3 **R**=3 **Ft**=4 · **MED risk — isolate in its own branch; full diff before merge.**

## Description
`zod-to-json-schema@3.25.2` is **unmaintained since Nov 2025** and incompatible with Zod 4. Zod 4 (stable since
2025) ships **native** `z.toJSONSchema()` and is 7–14× faster / ~57% smaller. Migrating drops the dead dep and
removes the benign-but-noisy `Recursive reference detected …offerWhen…` warnings. **It is broad** (touches every
schema file + the two JSON-Schema call sites + the golden master), so it is a deliberate, isolated migration.
(Research: [Zod v4 changelog](https://zod.dev/v4/changelog), [native JSON Schema](https://zod.dev/json-schema),
codemod [`zod-v3-to-v4`](https://github.com/nicoespeon/zod-v3-to-v4).)

## Acceptance Criteria
- `zod` upgraded to v4; **`zod-to-json-schema` removed** from `package.json` and the lockfile.
- The **two** JSON-Schema call sites use native `z.toJSONSchema(...)`:
  - `packages/content-schema/src/json-schema.ts` (`toJsonSchema()` for `ContentPack`),
  - `tools/pipeline/src/llm/adapter.ts` (`generateStructured`'s per-schema derivation).
  Set `unrepresentable:"any"` (+ `override` if needed) so it doesn't throw on any unrepresentable nodes.
- `generateStructured` **still returns `z.output<S>`** and default-field inference still holds — audit every
  `.default(...)` whose value differs by input/output type and switch to **`.prefault(...)`** where the *input*
  default is intended (this was a real inference fix point historically; verify the cycle's typed outputs).
- All existing tests pass unchanged in intent; the exported `ContentPack` JSON Schema is **diffed before/after**
  and is semantically equivalent (field names, required, enums) — attach the diff summary to the commit.
- **Golden-master** (`cycle.test.ts`) updated deliberately *iff* the change alters generated bytes; explain why.
- The `Recursive reference detected` stderr warnings are gone (or explained). `pnpm verify` fully green.
- Done on a dedicated branch; **not merged until green + reviewed**.

## Implementation approach
Isolate: `git worktree`/branch `spec/SPEC-16-zod4`. Run the `zod-v3-to-v4` codemod across `packages/content-schema/src/*`
and any other `zod` importers (grep `from "zod"`). Manually review: unified `error` param (drop
`message`/`invalid_type_error`), `.default()`→`.prefault()` audit, `z.string().email()`→`z.email()` if present,
`z.lazy` recursive `Condition` (re-verify it still type-checks under v4). Replace both `zodToJsonSchema(...)`
calls with `z.toJSONSchema(...)`. Run `pnpm schema:export` and diff against the prior output. Run full `pnpm
verify`; reconcile the golden hash last.
