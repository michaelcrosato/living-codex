# SPEC-61 — Vite 7 → 8 (Rolldown) upgrade

**Wave:** Cycle-7 / C7-P1 (dependency modernization — the deferred major is now unblocked). **Risk:**
LOW-MED (major version + bundler swap, but this repo uses none of the breaking-change config surfaces).
Reversible (lockfile + one version string; revert if gates fail).

## Description + Impact
BLOCKED.md deferred Vite 7→8 because "v8 didn't exist yet / no forcing function / Rolldown not 1.0." **That
changed:** Vite 8.0 shipped stable (2026-03-12), Rolldown-powered (single Rust bundler, 10–30× faster builds),
and **Vitest 4.1 (this repo is on 4.1.7) fully supports Vite 8** — the recommended migration order (bump
Vitest first, then Vite) is already half-satisfied. Staying on a soon-EOL major is the anti-pattern the
Cycle-2/3 modernization theme exists to prevent; the upgrade brings faster builds + future-proofing.

**Why low-risk here specifically:** `packages/app-web/vite.config.ts` is minimal (`root`, `server.fs.allow`,
`preview.port`, `build.outDir`) and `vitest.config.ts` uses only `vite-tsconfig-paths` + v8 coverage. The repo
uses **none** of Vite 8's breaking-change surfaces — no `rollupOptions.output.manualChunks` (→ codeSplitting),
no `optimizeDeps.esbuildOptions` (→ rolldownOptions), no custom minify options. Plugins (`vite-tsconfig-paths`)
work out of the box on Rolldown. The only residual risk is CJS-interop (escape hatch:
`legacy.inconsistentCjsInterop`) — but the app is ESM + JSON imports.

## Files (in scope)
- `packages/app-web/package.json` (`vite` `^7.0.0` → `^8.0.0`).
- `pnpm-lock.yaml` (regenerated by install).
- `pnpm-workspace.yaml` only if the `allowBuilds: esbuild` entry needs adjustment (Rolldown may drop esbuild).
- `packages/app-web/vite.config.ts` only if a breaking change actually surfaces (expected: none).

## Out of scope
- `@types/node` 24→25 (separate BACKLOG item, deliberately pinned to the runtime).
- Adopting `rolldownOptions`/`codeSplitting` tuning speculatively (no manual chunking today; GOAL §3).
- Vitest changes (already on 4.1.7, Vite-8-ready).

## DoD + Acceptance
- [ ] `vite` resolves to 8.x; `pnpm install --frozen-lockfile`-clean afterward.
- [ ] `pnpm verify` green (typecheck/lint/deps/test/content/replay) — Vitest 4.1.7 runs the suite on Vite 8.
- [ ] `pnpm --filter @codex/app-web build` green (Rolldown bundle); the app still boots.
- [ ] `pnpm e2e` 4 passed (the real Rolldown-built app boots + the cold-open slice + a11y specs pass).
- [ ] `pnpm audit` clean. No `vite.config.ts` breaking-change edit needed (or, if one is, it's documented here).
- [ ] If any gate fails and isn't trivially fixable → **revert cleanly** (restore lockfile + version), log the
      finding to BLOCKED.md/BACKLOG, mark `[!]`, take the next task (AFK guardrail: reversible, never halt).

## Test strategy
The existing full gate IS the test: `pnpm verify` exercises typecheck/lint/deps + the 264-test Vitest suite
on Vite 8; `build` proves the Rolldown bundle compiles; `e2e` proves the built app boots and plays in a real
browser. No new test needed — this is a toolchain swap whose contract is "everything still passes." Record the
new build output (chunk layout/size may shift under Rolldown) in the commit/JOURNAL for the bundle baseline.
